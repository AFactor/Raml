http://pipeline.tesco.com/v1.0/{region}/

Home

Digital Content Pipeline will call into the Digital Content API to fetch the marketing content and assets that will be rendered on the UI. Examples:

  1. Based on Asset ID: http://pipeline.tesco.com/v1/gb/digitalcontent?id=ASSET_ID_00011+ASSET_ID_11234+ASSET_ID_998034

  2. Query based on Asset metadata: http://pipeline.tesco.com/v1/gb/digitalcontent?metadata=title+summary&queryFilter=category:products

  • Response

Headers

  • Proxy-Authorization: (string)

    Example:

    d2b2a2a8-2adc-4492-88af-774df475b9fd

  • user-agent: (string)

    Example:

    Mozilla/5.0 (iPad; CPU OS 7_0 like Mac OS X) AppleWebKit/537.51.1 (KHTML, like Gecko) Version/7.0 Mobile/11A465 Safari/9537.53

  • Accept-Language: (string)

    Example:

    en-GB

  • Accept: (string)

    Example:

    text/json

  • Authorization: (string)

    Example:

    none if annonymous user journey.

Query Parameters

  • pageSize (integer)

    The number of results per page

    Example:

    20

  • offset (number)

    The number of results to skip

    Example:

    20

  • id required (string)

    Asset Ids separated by + delimiter

    Example:

    ASST_ID_00011+ASST_ID_11234+ASST_ID_998034

  • customerSegment (string)

    Depending upon the customer segment the digital content will vary.

    Example:

    Regular

  • metadata (string)

    Metadata filters, separated by +, to be applied when searching for the digital content/assets.

    Example:

    title+summary+thumbnail

  • queryFilter (string)

    Query filters to be applied when searching for the digital content/assets.

    Example:

    q=category:product

  • deviceFamily (string)

    Device family

    Example:

    PC, Mobile, Tablet, etc.
